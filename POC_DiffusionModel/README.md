# Quantum-Conditioned Diffusion POC

This folder contains a small proof of concept for the iQuCodeFest 2026
Quantum Image Generation track.

## Idea

The project demonstrates a tiny text-conditioned image generator using the
well-known `sklearn.datasets.load_digits` dataset.

Example prompts:

```text
generate digit 0
generate digit 3
generate digit 8
```

The system generates `32x32` colorized digit and shape images.

## Why This Fits The Challenge

The challenge asks for a working image-generation pipeline where the quantum
component has a clear role. This POC keeps that role narrow and testable:

```text
Text prompt
  -> digit label
  -> quantum circuit conditioned by that label
  -> measured latent mask
  -> tiny diffusion-like denoising process
  -> generated 32x32 color image
```

The quantum circuit is used as a probabilistic latent sampler. It produces an
`8x8` latent mask that is upsampled to the `32x32` image grid. For practical
runtime, the latent is generated by small row-wise quantum circuits instead of
one huge 64-qubit simulation. The measured quantum latent contributes more
strongly to the denoising target than the classical baseline.

## Dataset

The POC uses the built-in `sklearn` Digits dataset:

- Standard, known dataset
- No network download required
- Original `8x8` grayscale images, upscaled to `32x32` for this POC
- Labels from `0` to `9`
- Small enough for quantum-assisted simulation

## Run

From the repository root:

```bash
python POC_DiffusionModel/quantum_diffusion_poc.py
```

## Interactive Prompt Mode

To use it like a small terminal app:

```bash
python POC_DiffusionModel/prompt_terminal.py
```

Then type a prompt:

```text
generate digit 7
```

or:

```text
generate a triangle
draw a star
make a circle
yellow star
orange circle
blue triangle
```

Supported local prompt targets:

```text
digits: 0-9
shapes: circle, square, triangle, diamond, star, cross, plus, x
colors: blue, red, green, orange, purple, cyan, pink, lime, yellow, brown, white
```

The generated files are saved in:

```text
POC_DiffusionModel/outputs/terminal/
```

## Chat UI Server

The file `POC_DiffusionModel/test.py` is the Flask server used by the chat UI.
It exposes the same endpoints as the original root `test.py`:

```text
GET  /ping
POST /generate
```

Run it from the repository root:

```bash
python POC_DiffusionModel/test.py
```

Then open the UI and send prompts such as:

```text
yellow star
orange circle
blue triangle
generate digit 7
```

The server returns a PNG directly, so the UI pastes the generated image into
the chat as the assistant response.

If the prompt is not a known digit or supported shape, the server now falls
back to an abstract prompt-conditioned image. For example:

```text
neon quantum waterfall
purple spiral portal
angry electric banana
```

Those arbitrary prompts are not semantically understood yet, but they do
produce deterministic prompt-conditioned quantum images instead of failing.

Generated UI outputs are also saved in:

```text
POC_DiffusionModel/outputs/ui/
```

## Larger Dataset Options

The current shape mode is synthetic and local, so it does not require a
download. If you want a larger real dataset later, good options are:

- Google QuickDraw dataset: very large hand-drawn sketch dataset with classes
  such as `circle`, `square`, and `triangle`.
  https://github.com/googlecreativelab/quickdraw-dataset
- Hugging Face `google/quickdraw`: easier Python loading path for QuickDraw.
  Search for: `Hugging Face google quickdraw dataset`
  https://huggingface.co/datasets/google/quickdraw
- Sketchy dataset: object sketch dataset useful if you want object categories
  beyond simple shapes.
  Search for: `Sketchy dataset object sketches`
  https://huggingface.co/datasets/DrRORAL/sketchy-dataset
- dSprites: official/simple procedural shape dataset with square, ellipse, and
  heart latent factors.
  https://www.tensorflow.org/datasets/catalog/dsprites
- Kaggle 2D geometric shapes datasets: useful for circle/square/triangle and
  multi-shape experiments.
  https://www.kaggle.com/datasets/khalidboussaroual/2d-geometric-shapes-17-shapes

For the hackathon, synthetic local shapes are probably the fastest demo path.
QuickDraw is the better next step if you want the prompts to feel more like
real sketch generation.

Useful search terms:

```text
google quickdraw dataset 345 categories
huggingface google quickdraw image dataset
sketchy dataset object sketches
simple icons image dataset labels
geometric shapes image dataset rgb
captioned image dataset small objects
```

Custom prompts:

```bash
python POC_DiffusionModel/quantum_diffusion_poc.py --prompts "generate digit 1" "generate digit 4" "generate digit 9"
```

Useful options:

```bash
python POC_DiffusionModel/quantum_diffusion_poc.py --shots 1024 --steps 30 --seed 11
```

## Outputs

The script creates:

```text
POC_DiffusionModel/outputs/quantum_diffusion_report.png
POC_DiffusionModel/outputs/metrics.json
```

The report compares:

- Dataset prototype
- Classical baseline output
- Quantum latent mask
- Quantum-assisted output

The metrics file stores MAE/MSE and classifier predictions for each prompt.

## Current Limitations

This is not a full production diffusion model. It is a compact hackathon
prototype designed to show the core mechanism:

- Text prompts are parsed into digit labels.
- The denoising process is educational and lightweight.
- The quantum circuit is simulated with Qiskit Aer.
- The generated image resolution is currently `32x32`.

Future work could replace the simple denoising rule with a trained denoiser,
scale to MNIST, or test different quantum feature maps and circuit depths.
